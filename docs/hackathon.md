# Pacifica Hackathon — Rules & Logistics

## Timeline

| Date | Phase |
|------|-------|
| Feb 25, 2026 | Announcement + welcome AMA |
| **Mar 16 – Apr 16, 2026** | **Dev period** (one month) |
| Apr 16, 2026 | Submission deadline |
| Post-deadline | Demo day + judging |

## Tracks

1. **Trading Applications & Bots** — algo trading, execution optimization
2. **Analytics & Data** — market intel, PnL tracking, risk dashboards
3. **Social & Gamification** — leaderboards, copy trading, competitions
4. **DeFi Composability** — building on Pacifica perps infra

## Team Rules

- Solo or up to 5 members.
- Register via official form + join Discord.
- Team members changeable only in the first half of the hackathon.
- **MUST use Pacifica API and/or Builder Code.**
- **All work must be created during the hackathon window.** Pre-existing projects are not eligible. External code must be clearly acknowledged.
- Submission is per team, by the deadline.

## Judging Criteria

| Criterion | What it means |
|-----------|---------------|
| Innovation | Novel / creative use of Pacifica infra |
| Technical Execution | Code quality, API integration, rigor |
| User Experience | Polish, usability, intuitive design |
| Potential Impact | Likelihood real traders adopt it |
| Presentation | Clarity of demo |
